# План действий Toxic Scanner

Обновлено: 2026-07-07. Логика плана: сначала доказать спрос, потом
договариваться и брать деньги. До подтверждения спроса — никаких
взаимодействий с третьими сторонами (заявки, переговоры, покупка API).

## Сделано (v0.1–v0.8)

- Резолвер компании: NIP / KRS / название (Tavily + верификация в
  реестрах, кандидата подтверждает пользователь).
- Жёсткие факты из одписа KRS: возраст, капитал, PKD, дисциплина
  отчётности (dział 3), задолженности (dział 4), ликвидация (dział 6).
- Финансовый адаптер `app/sources/financials.py`: выручка/прибыль/
  капитал/обязательства по годам (okredo основной, aleo fallback,
  через Jina Reader). Подключён в отчёт и evidence.
- Health-check отчёт: «О компании» (PKD + сайт + новости), финтаблица
  из реальных цифр, отзывы GoWork/Reddit, риск-флаги, чеклист.
- Звезда качества, 5 осей: Надёжность/Финансы/Динамика/Прозрачность —
  детерминированные формулы, Люди — LLM. Оси без данных — «н/д».
- UI: HTML-рендер отчёта (marked+DOMPurify), радар, evidence JSON.

## Шаг 1. Лимиты + телеметрия (перед деплоем)

- Деплой на Cloud Run `min-instances=0` (засыпает до нуля → на объёме
  валидации по сути бесплатно). Always-on и кэш — только если холодный
  старт реально мешает на живой ссылке.
- Кэш отчётов по KRS — в беклог (нет масштаба; при `min-instances=0`
  файловый кэш всё равно стирается холодным стартом).
- Rate-limit на IP для `/companies/*` — предохранитель от неограниченного
  счёта за Gemini/Tavily (стоимость на запрос реальная, не про масштаб).
- Учесть дневные лимиты wl-api.mf.gov.pl (белый список VAT).
- Телеметрия для валидации спроса — PostHog Cloud EU (free tier):
  ключевые события слать server-side из FastAPI (search_started,
  report_built, report_cache_hit), distinct_id = хэш IP — аудитория
  Reddit/Wykop сидит под адблоками, клиентский JS занизит цифры;
  лёгкий JS-сниппет опционально, только для глубины чтения отчёта.
  UTM-метки на ссылки: отдельные на канал и на упаковку
  (работодатель vs контрагент).
- Fake door готовности платить (СДЕЛАНО, копирайт русский — см. вопрос
  языка): после N бесплатных отчётов с IP (per-IP лимит) — экран
  «пакет из 10 отчётов за 20 zł», кнопка «купить» (сигнал pay_click) +
  «оставить email». Оффер через env: REPORT_PACK_SIZE / REPORT_PRICE_PLN.
  События сейчас в лог (Cloud Logging), PostHog подключится в тот же
  seam track(); email — в отдельный вейтлист-сток.

## Шаг 3. Валидация спроса (единственный вопрос сейчас)

- Раздать ссылку руками: Reddit (r/Polska и пр.), Wykop, LinkedIn,
  группы «praca w IT». SEO-заготовка: «czy firma X jest wiarygodna».
- Смотреть: возвращаются ли, докуда доходят, что спрашивают.
- Критерии решения зафиксировать ДО постинга ссылки. Черновик:
  окно 2 недели после постинга в 3–5 местах; спрос есть, если
  ≥30 построенных отчётов от ≥15 уникальных посетителей И есть
  возвраты в другой день ИЛИ email/клики на fake door. Активаций ~0
  при живом трафике — менять упаковку, не продукт; нет трафика —
  проблема дистрибуции. Ретеншен судить только по упаковке
  «контрагент»: соискатель по природе разовый.
- Две упаковки одной механики: «проверь работодателя перед оффером»
  (соискатель, разовая) и «проверь контрагента» (фрилансер/малый
  бизнес, повторяющаяся — потенциально сильнее).

## Шаг 4. Только если спрос подтверждён

  серая зона, для платного продукта нельзя; карта в памяти проекта,
  файл legal-sources): wniosek в Минюст на повторное использование RDF
  (закон 11.08.2021, госAPI KRS уже CC0) ИЛИ платный API
  (Transparent Data / MGBI / bizraport / okredo). Вопрос недель,
  не блокер.
- Монетизация без платёжного бэкенда: 1–2 бесплатные проверки,
  дальше Stripe Payment Link / Przelewy24 за разовый отчёт.
